Lesson 1: The Basics of the Web and HTML

How the Web Works

The web serves as a bunch of computers with many pieces all of which communicate with each other. Users initiate a requests via
an HTTP Request that interacts with a server. Udacity Intro to Programming offers this video does a good job of explaining more.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web.

There are approximately 30 billion pages and hyperlinks are throughout. A variety of document types and hyperlinks included within HTML. These include:

  • Text, which is the content (what you see).
  • Markup, what the content looks like.
  • URLs, referenes to other documents, i.e. images and videos.
  • Links to other pages.

Udacity offers this video providing an overview

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. Udacity's video explains the distinction well.

Tags are primarily composed of opening and closing tags. However, there are tags called 'void' which do not require a closing tag.

Failing to properly close a tag may result in everything following the opening tag to assume that attribute of that tag. For example, if one were to use an opening tag of "em" and fail to close that tag, everything from the opening tag forward would be shown in italics.

Sample tags include:

  • b, for bolding text between opening and closing tags
  • em, for italicizing text
  • br, creates a break between text.
  • p, paragraph, does just that. LOL. It has content.

Attributes are properties of HTML and used properly, create powerful pages.
Attributes include:

  • HREF, which creates a link to access additional information/site. For example: Udacity
  • IMG, which does not contain a closing tag. It is considered a "void" tag. In the example below the use of 'alt' creates an alternate view of the referenced source. This is quite helpful should a web error occur and supports a blind computer user.'

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Inline elements include:

  • a, used to create links such as derp Opening and closing tags used.
  • br, used toens a line and subsequently wraps text to next line. No closing tag.
  • em, emphasizes and italicizes words. Opening and closing tags used.
  • img, introduces a picture. No closing tag
  • span, uses opening and closing tags
  • strong, uses opening and closing tags

Block elemeents include:

  • p, which creates in invisible "block" around everything included within opening and closing tags. Additional attributes such as height and width may also be applied.
  • form and its closing tags where developer is creating a form for a user to complete.

Addtionally these block elements may be considered 'container' elements and have content.

  • div and its closing tag. Syntax::
    text
  • span and its closing tag. Syntax: text , where ‘class’ is a CSS style.

Lesson 3: Adding CSS Style to HTML Structure

Creating a Structured Document with HTML

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

Benefits of Style Sheets:

Avoiding repetition creates cleaner code which may then become 'reuseable'. It supports "work smarter, not harder".

Creating consistency: Use CSS (Cascading Style Sheets) to standardize the look of web pages.
